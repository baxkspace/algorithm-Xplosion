## BOJ 1021 회전하는 큐 KDH

## 문제 요약
- 1~N까지의 숫자(또는 임의의 원소)가 순서대로 큐에 들어와있다.
- 해당 큐에서는 첫번째 원소를 뽑거나(1), 시계방향으로 한 칸 회전하거나(2), 반시계방향으로 한 칸 회전시킬 수 있다(3)
- 1, 2, 3번 operation을 적절히 사용해서 주어진 M개의 숫자들을 차례로 큐에서 뽑아내야 한다.
- 단, 2, 3번 동작의 횟수를 최소한으로 가져가야 한다.

## 풀이 방법
- 회전 큐이고, 원소 pop 함수도 필요하다. 순환 큐를 구현해서 사용해도 되지만, deque를 사용했다.
- deque는 연속적이지 않은 메모리를 할당받아서 double linked list처럼 동작한다. deque의 앞과 뒤에서 원소의 추가, 삭제가 모두 O(1)으로 이루어진다.

- 항상 원소의 추출은 큐의 첫번째 자리에서만 가능하다. 큐의 두번쨰 원소를 첫번째 자리에 위치시키는 동작을 반시계 방향으로 1회 회전시키는 3번 과정이라고 가정한다.
- 큐의 첫번째 원소 자리에서 시작해, 다음으로 추출해야 하는 target 원소 자리까지는 시계 방향으로 큐를 움직일 수도 있고, 반시계 방향으로 큐를 움직일 수도 있다. 
- 항상 반시계 방향으로 움직였을 때 걸리는 회전 횟수를 활용해 시계 방향으로 돌렸을 때 걸리는 회전 횟수도 함께 구한다. (반시계 방향 회전 횟수 + 시계 방향 회전 횟수 = 전체 큐의 원소 개수)
- 반시계 방향 회전 횟수, 시계 방향 회전 횟수 중 적은 값을 반복해서 선택하고 `result`에 누적해서 더한다.
- `result` 값을 최종 출력한다.
